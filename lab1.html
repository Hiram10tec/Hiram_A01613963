Preguntas de laboratorio 1
1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
Internet: Es la red global de redes de computadoras interconectadas. Permite la comunicación y el intercambio de información entre dispositivos ubicados en diferentes partes del mundo.
World Wide Web (WWW):Es un servicio específico de internet que permite acceder y visualizar documentos multimedia (páginas web) a través de enlaces. La web es solo una parte de internet.
2. ¿Cuáles son las partes de un URL?
Protocolo: Especifica cómo se debe acceder al recurso (por ejemplo, "http" o "https").
Dominio: La dirección única que identifica un sitio web.
Ruta: La ubicación específica del recurso en el servidor.
Parámetros: Información adicional, a menudo utilizada en la consulta de una base de datos.
Fragmento: Una referencia a una sección específica dentro del recurso.
3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
GET: Solicita datos del servidor.
HEAD: Similar a GET, pero solo solicita encabezados, no el cuerpo del recurso.
POST: Envía datos al servidor para ser procesados (utilizado comúnmente en formularios).
PUT: Actualiza un recurso existente en el servidor.
PATCH: Modifica parcialmente un recurso existente.
DELETE: Elimina un recurso en el servidor.
4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML? ¿Por qué?
Se utiliza POST para enviar datos al servidor de manera segura y confiable, como al enviar información de un formulario. Los datos no se muestran en la URL y pueden contener información sensible.
5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
GET: Cuando solicitas una página web, estás pidiendo datos al servidor para ser mostrados en tu navegador.
6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
El código 200 significa "OK". La solicitud fue exitosa y el servidor devuelve la respuesta solicitada.
7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Un error 404 indica que la página no fue encontrada en el servidor. Si es responsabilidad del desarrollador asegurarse de que los enlaces estén actualizados y apunten a páginas existentes, aunque en algunos casos esta puede ser una tecnica para prevenir hackeos, haciendo creer que el archivo no existe, así como lo utiliza github con repositorios privados.
8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Un error 500 significa problema interno en el servidor. El desarrollador debe corregir el problema para que el sitio funcione de manera normal.
9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
Deprecated significa que el atributo o elemento ya no se recomienda para su uso y puede ser eliminado en futuras versiones.
Algunos elementos de HTML 4 desaprobados en HTML5 son: font, center, y strike.
10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
HTML5: Introduce nuevas etiquetas semánticas (como header, footer, nav), soporte para multimedia sin necesidad de plugins (como audio y video), y mejoras en la manipulación de contenido dinámico.
11. ¿Qué componentes de estructura y estilo tiene una tabla?
Estructura: table, td (celda), tr (fila), th (encabezado de celda), thead, tbody, tfoot.
El estilo puede ser aplicado con atributos HTML, por ejemplo: style. O mediante hojas de estilo CSS para personalizar colores, bordes, espaciado, etc.
12. ¿Cuáles son los principales controles de una forma HTML5?
input: Para entradas de texto.
textarea: Para áreas de texto más extensas.
select: Para listas desplegables.
button: Para botones

14. ¿Cuál es el ciclo de vida de los sistemas de información?
El ciclo de vida de los sistemas de información es:
Definición de necesidades: Identificación de problemas o necesidades a abordar.
Planificación: Desarrollo de un plan para abordar esas necesidades.
Diseño: Creación de una solución basada en el plan.
Implementación: Puesta en marcha de la solución.
Mantenimiento y evaluación: Monitoreo y ajuste continuo para garantizar la efectividad y la adaptación a cambios.
15. ¿Cuál es el ciclo de desarrollo de sistemas de información?
Investigación y análisis:Identificación y comprensión de los requisitos del sistema.
Diseño: Creación de la estructura y la arquitectura del sistema.
Desarrollo: Implementación del sistema basado en el diseño.
Pruebas: Verificación de que el sistema funcione según lo previsto.
Implementación: Puesta en marcha del sistema en un entorno de producción.
Mantenimiento: Actualizaciones, correcciones de errores y mejoras continuas.
